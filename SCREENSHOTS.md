## üé® Screenshot Design Best Practices

### Text Overlays
**Recommended Tools:**
- Figma (free, professional)
- Canva (easy, templates)
- Photoshop (advanced)
- Screenshot Frame (iOS app)
- AppLaunchpad (paid service)

**Text Overlay Guidelines:**
- **Font:** Bold, sans-serif (Helvetica, Arial, Inter, SF Pro)
- **Size:** Large enough to read on small devices (36-48pt)
- **Color:** White text with dark shadow/background for contrast
- **Position:** Top or bottom third of screen (not covering key UI)
- **Length:** 1-2 short lines max (5-10 words)
- **Background:** Semi-transparent dark overlay (40-60% opacity) behind text

### Device Frames (Optional)
**Should You Add Device Frames?**
- ‚úÖ **YES** if screenshots look "floating" or need context
- ‚úÖ **YES** for marketing/website (looks more polished)
- ‚ùå **NO** for App Store if showing full-screen content (Apple discourages)
- ‚ùå **NO** for Play Store (Google discourages device frames)

**If Using Frames:**
- Use realistic device mockups (iPhone 15 Pro, Pixel 8)
- Match actual device dimensions
- Don't use outdated device frames

### Screenshot Order
**Best Practice Order:**
1. **Core Feature First:** Home screen with session logging
2. **Value Proposition:** Stats/analytics showing insights
3. **Unique Features:** Badges, recovery mode, mood tracking
4. **Visual Appeal:** Dark mode or beautiful UI shots
5. **Trust/Privacy:** Settings showing privacy features

### Common Mistakes to Avoid
‚ùå **Empty States:** Never show empty charts or "No data" screens
‚ùå **Lorem Ipsum:** Don't use placeholder text
‚ùå **Unrealistic Data:** Don't show 100 sessions in one day
‚ùå **Broken UI:** Ensure no visual bugs or cut-off text
‚ùå **Old Versions:** Update screenshots with every major UI change
‚ùå **Low Quality:** Never upscale lower-res images
‚ùå **Wrong Aspect Ratio:** Match exact pixel dimensions required

## üõ†Ô∏è How to Generate Screenshots

### Method 1: Simulator/Emulator (Recommended for iOS)

**iOS (Xcode Simulator):**
```bash
# Install Expo CLI if not already installed
npm install -g expo-cli

# Start the development server
npx expo start

# Press 'i' to open iOS Simulator
# OR manually: Xcode ‚Üí Open Developer Tool ‚Üí Simulator

# Choose device:
# - iPhone 15 Pro Max (6.7" - REQUIRED)
# - iPhone 11 Pro Max (6.5" - REQUIRED)

# Take screenshot:
# CMD + S (saves to Desktop)
# Or: File ‚Üí New Screenshot

# Verify dimensions in Finder
```

**Android (Android Studio Emulator):**
```bash
# Start development server
npx expo start

# Press 'a' to open Android Emulator
# OR manually open Android Studio ‚Üí AVD Manager

# Choose device:
# - Pixel 7 Pro (1440 x 3120)
# - Pixel 5 (1080 x 2340)

# Take screenshot:
# Emulator toolbar ‚Üí Camera icon
# Or: Run ‚Üí Take Screenshot

# Screenshots saved to: ~/Desktop or configured location
```

### Method 2: Physical Device (Best for Realistic Look)

**iOS:**
1. Build app to physical device using Expo Go or development build
2. Navigate to desired screen
3. Take screenshot: **Press Volume Up + Side Button simultaneously**
4. Screenshots save to Photos app
5. AirDrop to computer or sync via iCloud Photos

**Android:**
1. Build app to device
2. Navigate to desired screen
3. Take screenshot: **Press Power + Volume Down simultaneously**
4. Screenshots save to Gallery/Photos
5. Transfer via USB or Google Photos sync

### Method 3: Screenshot Automation Tools

**Fastlane Snapshot (iOS):**
```ruby
# Install Fastlane
gem install fastlane

# Set up Snapshot
fastlane snapshot init

# Configure Snapfile for devices and languages
# Run automated screenshot capture
fastlane snapshot
```

**Screengrab (Android):**
```bash
# Install Screengrab
gem install screengrab

# Configure Screengrabfile
# Run automated screenshot capture
fastlane screengrab
```

## üöÄ Final Screenshot Export Settings

### File Format
- **iOS:** PNG or JPG (PNG recommended for sharp UI)
- **Android:** PNG or JPG (24-bit, no transparency)

### Color Profile
- **sRGB** (standard for web/mobile)
- Not Adobe RGB or P3 (causes color shifts)

### Compression
- **Quality:** 90-100% (don't over-compress)
- **Max File Size:** 5MB per screenshot (both stores)
- **Tool:** TinyPNG, ImageOptim, or export from design tool

### File Naming Convention
```
ios-6.7-01-home-screen.png
ios-6.7-02-session-entry.png
ios-6.7-03-usage-stats.png
```
